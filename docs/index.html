Work Experience

imgConvert

Programming - Single Page Application

My most recent, and most difficult to date, project was this piece of image conversion software. Built in Angular 19, I had many different needs to consider. My stakeholder needed a piece of software that would let her convert regular images of clip art that she had found into scalable vector graphics (.svg). While there were already tools out there that could perform this task, I thought I would also try figuring this problem out.



When considering the framework to use, I wanted to use a reasonably modern front-end framework so I could learn as much relevant knowledge as possible. After a bit of research I settled on using Angular, and if necessary, I would support it with an express server for image processing. I had to weigh the benefits of providing all the code to the front-end, vs supported with a service (either local or remote), which ended up boiling down to how well I could get image processing to work whilst in the constraints of a browser window. In the end, I was able to provide a satisfactory level of performance without server support, so all the functionality is provided to the end user. An additional advantage was that security was much less of a concern. There weren't many vectors for malicious behaviour.

Overlays

Programming - Graphical Overlay

I initially started this project in c#, as a very narrow scoped fuel calculation service. After a few iterations, I settled on Node.js as the architecture to use. Learning to program in javascript was fairly natural to me, having learned a decent amount in the 2 terms of post-secondary i completed. Having began anew, I had started with a relative positioning overlay, and with a rather limited understanding of how iRacing's telemetry was formatted. This program really required a lot of trial and error to find the correct properties in the iRacing API, but I continued on trying new techniques and observing these objects I could not control.



Each overlay is a simple, borderless Electron browser. This made it easy to access remotely if desired, debug using a typical browser and make transparent and clickthrough, which are critical for gaming overlays. At several times during the development of each overlay I would discover some new way of building them and test them. I now consider this project complete, and, with the help of people who know about it, track issues and push improvements. One of the last improvements was switching the stylesheets to use relative sizing, and simply controlling the size of the window it was contained in. This required much less script, held much less error potential, and was much quicker to respond when initializing and resizing windows. I am really happy with how this turned project turned out.

Woodland Supply

Machining - Wood Products

Precision machining was the main priority of what I did here. My job was to create trimming for doors, windows, ceiling edges, and more. There were tight tolerances that had to be maintained for dimensions, and strict quality control for natural wood defects that had to be enforced. In order to maintain these requirements, a high level of training, precise run-time adjustments, and judgement was necessary Appropriate cutting tools needed to be selected to create the profile required by any order. Any small chipping, burning, or cracking had to be sanded and patched, while larger defects were unsuitable for shipping, and had to be discarded.

Dewitt Bros. Foundations

Construction - Concrete Foundations
