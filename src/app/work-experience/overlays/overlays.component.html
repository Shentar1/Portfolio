I initially started this project in c#, as a very narrow scoped fuel calculation service. After a few iterations, I settled on Node.js as the architecture to use. Learning to program in javascript was fairly natural to me, having learned a decent amount in the 2 terms of post-secondary i completed. Having began anew, I had started with a relative positioning overlay, and with a rather limited understanding of how iRacing's telemetry was formatted. This program really required a lot of trial and error to find the correct properties in the iRacing API, but I continued on trying new techniques and observing these objects I could not control.



Each overlay is a simple, borderless Electron browser. This made it easy to access remotely if desired, debug using a typical browser and make transparent and clickthrough, which are critical for gaming overlays. At several times during the development of each overlay I would discover some new way of building them and test them. I now consider this project complete, and, with the help of people who know about it, track issues and push improvements. One of the last improvements was switching the stylesheets to use relative sizing, and simply controlling the size of the window it was contained in. This required much less script, held much less error potential, and was much quicker to respond when initializing and resizing windows. I am really happy with how this turned project turned out.
